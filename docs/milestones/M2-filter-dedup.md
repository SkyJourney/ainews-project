# M2 — 过滤/去重规则完整化

> 前置依赖：[M1-single-source-e2e.md](./M1-single-source-e2e.md)
> 状态：未开始
> 关联文档：[04-roadmap.md](../04-roadmap.md) §2.3（本里程碑的权威规则来源）· §4 M2

## 目标

把 `filter_activity` 从 M1 的最简版补全为 04 §2.3 定义的完整规则集。

## Scope（范围内）

完整实现 04 §2.3 全部规则：执行顺序、同批次去重四级优先、跨源同论文去重、时效过滤、信噪比过滤、模糊地带兜底、跨日去重 30 天索引、统计自检。

## Out of scope（明确不做）

- 多源接入（留 M3——本里程碑仍在 `openai-rss` 单源上验证规则正确性即可，也可以提前用多源数据测试去重效果，但不强制）
- Enrich / Aggregate 阶段的规则（留 M4/M5）

## 任务清单

- [ ] 执行顺序校正：同批次去重 → 时效过滤（先于信噪过滤）→ 信噪过滤 → 跨日去重
- [ ] 同批次去重补全 4 级优先（命中即判重复，保留信息最完整一条）：① URL 完全相同 ② URL host+path 相同（忽略 query）③ 标题语义相似度 ≥0.85 ④ 摘要前 100 字重叠 ≥0.9
- [ ] 跨源同论文去重：范围限论文类源，标题归一化后精确相等即合并，保留规范链接源版本，其余来源并入 `also_reported_by`
- [ ] 不去重情形核查：中英双语同事件报道两条都保留（聚类阶段合并）；"通稿"与"深度分析"两种角度都保留
- [ ] 时效过滤：单一阈值，发布日期距今 >14 天一律丢弃（`stale:Nd_old`），不分来源等级
- [ ] 信噪比过滤表：丢弃类别（融资 PR / 招聘 / 活动通知 / 广告 / VC 软文 / 营销软文 / 二手编译）+ 保留优先信号（arxiv.org/github.com/huggingface.co 域名、benchmark/SOTA、开源/论文关键词、政策类、安全对齐类）——保留信号覆盖所有丢弃规则
- [ ] 模糊地带兜底：判断不清 → 保留 + 标 `low_confidence=true`，交聚类阶段二判；不在过滤阶段误删
- [ ] 跨日去重 30 天滚动窗口索引：URL 归一化函数（小写、去 scheme、去 www、去 query/锚点、去尾斜杠）
- [ ] 跨日去重命中规则：命中索引 ≤7 天默认丢弃，除非 Jaccard 词汇重叠 ≤0.6（新角度豁免，标 `re_coverage=true`）；>7 天视为已淡出正常保留
- [ ] Jaccard 重叠计算实现：中文按字切分去停用词，英文按空格切分小写去停用词，`overlap=|A∩B|/|A∪B|`，任一方摘要为空则 overlap=0（不豁免）
- [ ] 索引维护：每条 URL 节点含 `first_seen_date`/`first_seen_run`/`title`/`source_name`/`kept_in_daily[]`/`zettel_id`/`raw_summary_excerpt`；`first_seen_date` 超 30 天读取时清理
- [ ] 索引写权限约束：仅 Filter（读写全部字段）和 Write（仅回填 `zettel_id`）能碰这个索引，其余阶段严禁写入——实现时用代码层面的访问控制体现这条约束，不要只写在注释里
- [ ] 统计自检：`entries_after_dedup ≥ entries_after_filter ≥ 最终保留数`，各阶段丢弃计数之和应与总数对上

## 验收标准

- [ ] 统计自检（单调递减关系）跑通
- [ ] 能验证"复盘"条目正确复用旧 `zettel_id` 而不是重复创建

## 备注 / 风险

- 这一阶段的规则全部是"批量、纯规则、必须在 fan-out 前完成"——04 §2.3 强调去重本质是跨条目比较，不能拆到独立子线程里做，实现时不要为了并行化而破坏这个约束。
